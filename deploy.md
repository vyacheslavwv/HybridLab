# Развертывание проекта Hybrid Lab

Данный документ описывает процесс развертывания (деплоя) клиентской и серверной частей приложения, а также настройку инфраструктуры на базе Yandex Cloud.

## Архитектура и стек технологий
* **Инфраструктура:** Виртуальная машина Yandex Cloud (Ubuntu/Debian)
* **Оркестрация:** Docker, Docker Compose
* **Веб-сервер / Reverse Proxy:** Nginx
* **Безопасность:** SSL/HTTPS (Certbot / Let's Encrypt)
* **Управление доступом:** SSH, GitHub Deploy Keys

## Этапы развертывания

### 1. Подготовка сервера
* Настроено безопасное SSH-соединение с виртуальной машиной в Yandex Cloud.
* Установлены необходимые зависимости: `docker` и `docker-compose`.

### 2. Настройка доступов и репозитория
* Сгенерированы SSH-ключи (Deploy Keys) для безопасного клонирования приватного репозитория GitHub на сервер.
* Структура проекта оптимизирована для корректной работы пушей и сборки контейнеров.

### 3. Контейнеризация
* Разработаны `Dockerfile` для независимой сборки образов фронтенда и бэкенда.
* Фронтенд и бэкенд упакованы в изолированные Docker-контейнеры для обеспечения консистентности среды между разработкой и продакшеном.

### 4. Оркестрация через Docker Compose
* Создан `docker-compose.yml`, объединяющий микросервисы в единую сеть.
* Настроены переменные окружения, проброс портов и зависимости между контейнерами (например, запуск бэкенда после базы данных).

### 5. Настройка Nginx в качестве Reverse Proxy
* Nginx развернут для маршрутизации внешнего трафика.
* Настроено проксирование запросов: API-вызовы направляются на контейнер бэкенда, а запросы к статике — на контейнер фронтенда.

### 6. Защита соединения (HTTPS)
* Привязан домен к IP-адресу сервера Yandex Cloud.
* Использована утилита Certbot для автоматического выпуска и настройки SSL-сертификатов от Let's Encrypt.
* Nginx сконфигурирован для принудительного редиректа с HTTP на HTTPS.
